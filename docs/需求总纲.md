# 全能脚本 · 需求总纲

> 基于整个对话过程汇总

---

## 一、一句话定义

一个**两页切换**的游戏自动化脚本工具，支持**基础步骤模式**和**高级循环节点模式**，能自动判断地图特征并决策继续/重开。

---

## 二、两种页面

### 页面1：基础页（Basic）

参考现有的 tkinter 脚本功能，Web 化：

- 预设管理（加载/保存/删除预设）
- 步骤列表（添加/编辑/删除/排序步骤）
- 运行控制（▶开始 / ■停止）
- 条件 - 动作流程：
  - 条件：直接执行 / 文字识别 / 模板识别
  - 动作：按键 / 点击文字 / 点击图片 / 偏移点击 / 鼠标滑动
- 实时执行日志

### 页面2：循环节点页（Node/Loop）

类似 Kun 的节点编辑器，可视化编排流程：

- **地图选择** — 指定当前关卡需要检测哪些地图
- **特征节点** — 每张地图 4 个特征，每个特征可设：
  - `detect_type`：text（文字识别）/ image（模板匹配）/ key（按键检测）
  - `detect_value`：检测的目标值
  - `on_match`：匹配时 → continue（继续）或 restart（重开）
  - `on_mismatch`：不匹配时 → continue 或 restart
- **判定逻辑** — 4 个特征全部通过才继续，任一不满足则重开本关
- **循环控制** — 全图通过后自动循环下一轮

---

## 三、核心流程

```
关卡开始
  ↓
第1张随机地图出现
  ↓
检测地图4个特征节点（OCR/模板/按键）
  ├─ 全部匹配 → 继续
  └─ 任一不匹配 → 重开本关
  ↓
第2张随机地图出现
  ↓
检测地图4个特征节点
  ├─ 全部匹配 → 继续
  └─ 任一不匹配 → 重开本关
  ↓
全部地图通过 → 循环下一轮
```

## 四、技术规格

| 项目 | 规格 |
|------|------|
| **前端** | Web UI（HTML/CSS/JS） |
| **后端** | Python FastAPI（端口 8765） |
| **通信** | HTTP REST + SSE 实时日志 |
| **引擎** | 复用现有 engine.py（OCR/截图/模板匹配/输入模拟） |
| **节点引擎** | 新写 node_engine.py |
| **打包** | PyInstaller → 单 exe |
| **架构** | 前后端分离 |
