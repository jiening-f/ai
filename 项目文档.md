# 全能游戏脚本

> 创建日期：2025年

---

## 一、主要用途

通用游戏自动化脚本，支持两种操作模式：

**基础页** — 类似现有 tkinter 脚本，步骤列表式操作，适合简单固定流程  
**循环节点页** — 节点流程图式操作，适合有多张随机地图的关卡

---

## 二、核心概念

### 地图-节点系统

```
一个关卡 = 多张随机地图（每次进入随机出现 N 张）
                │
         用户指定哪些地图需要操作
                │
         每张地图有 4 个特征节点
                │
    节点1 → 节点2 → 节点3 → 节点4
        特征A    特征B    特征C    特征D
                │
         根据特征匹配结果决定：
         ├─ 全部匹配 → 继续执行
         └─ 任意不匹配 → 重开本关
```

### 节点编辑器（高级页）

```
┌──────────────────────────────────────────────┐
│  循环节点编辑器                                │
│                                               │
│  [地图选择节点] → [特征节点1] → [特征节点2]    │
│       │              │              │          │
│  指定要检测的地图    OCR识别的特征   模板匹配特征│
│                                               │
│  [特征节点3] → [特征节点4] → [判定节点]        │
│       │              │         ├─ 继续        │
│  按键测试特征   图像特征      └─ 重开        │
│                                               │
│  节点之间可以连线，自由编排逻辑流程               │
└──────────────────────────────────────────────┘
```

## 三、架构设计

```
┌──────────────────────────────────────────────────────────┐
│  前端 (Web UI)                                           │
│  ├─ 基础页 — 预设管理 + 步骤列表 + 运行控制 + 日志       │
│  ├─ 循环节点页 — 节点编辑器 + 地图管理 + 流程编排         │
│  └─ 页面切换器                                             │
├──────────────────────────────────────────────────────────┤
│  后端 API (FastAPI)                                       │
│  ├─ /api/presets       预设 CRUD                         │
│  ├─ /api/nodes         节点配置 CRUD                      │
│  ├─ /api/maps          地图配置 CRUD                      │
│  ├─ /api/run           启动运行                           │
│  ├─ /api/stop          停止                               │
│  └─ /api/logs/stream   SSE 日志流                         │
├──────────────────────────────────────────────────────────┤
│  引擎层 (Python)                                          │
│  ├─ engine.py          执行引擎（从脚本备份复制改造）       │
│  ├─ node_engine.py     节点引擎（新写）                    │
│  ├─ vision.py          OCR/截图/模板匹配                   │
│  └─ input_sim.py       按键/鼠标                          │
└──────────────────────────────────────────────────────────┘
```

## 六、关键接口定义

### 节点配置

```json
{
  "maps": [
    {
      "id": "map_1",
      "name": "第一关地图A",
      "enabled": true,
      "nodes": [
        {
          "id": "node_1",
          "type": "feature",
          "detect_type": "text",        // text / image / key
          "detect_value": "BUFF_A",
          "on_match": "continue",       // continue / restart
          "on_mismatch": "restart"
        },
        {
          "id": "node_2",
          "type": "feature",
          "detect_type": "image",
          "detect_value": "宝箱图标",
          "on_match": "continue",
          "on_mismatch": "continue"      // 没有宝箱也继续
        }
      ]
    }
  ],
  "flow": [
    "map_1_nodes",    // 先检测第一关地图的特征
    "map_2_nodes",    // 再检测第二关
    "loop"            // 全部通过后循环
  ]
}
```

### HTTP API

```
基础页接口：
  GET    /api/presets
  POST   /api/run/basic      # 运行基础预设
  POST   /api/stop

节点页接口：
  GET    /api/maps
  POST   /api/maps
  PUT    /api/maps/:id
  DELETE /api/maps/:id
  POST   /api/run/node        # 运行节点流程
  GET    /api/logs
  GET    /api/logs/stream     # SSE
```
